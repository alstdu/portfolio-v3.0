Challenges

Overcoming complex technical hurdles during development.

Performance

One of the biggest challenges was performance when loading the spells. It would take around 10 seconds to load the spells since we were loading all the spells at once.

To fix this, I created a custom caching system and implemented a progressive loading system. This way, the page would load faster and the user would be able to start using the tools sooner.

Spell Information

The problem I faced was choosing how the spell information would display on smaller screens. I couldn't fit everything on the table at smaller screen sizes, so I added a transition that drops less important information when the screen is small, and puts that information in the expanded section for that spell.
